Types of S3 Replication \
🌍 1. Cross-Region Replication (CRR) \
Replicates data to a different AWS region \
Used for: \
Disaster recovery \
Global access \
Compliance \
 \
👉 Example: \
 \
Source: Mumbai \
Destination: Singapore \
🏠 2. Same-Region Replication (SRR) \
Replicates within the same region \
Used for: \
Data isolation \
Log aggregation \
Compliance requirements \
🔹 How It Works \
Object uploaded to source bucket \
S3 automatically copies it to destination \
Replication happens asynchronously \
🔹 Key Requirements \
 \
To enable replication: \
 \
✅ Versioning must be enabled on both buckets \
✅ Proper IAM role for replication \
✅ Buckets must be different (cannot replicate to same bucket) \
 \
Advanced Features \
🔁 1. Replication Rules \
Define: \
Prefix (e.g., /images/) \
Tags (e.g., env=prod) \
Allows selective replication \
⏱️ 2. Replication Time Control (RTC) \
Guarantees replication within 15 minutes \
SLA-backed \
 \
👉 Important for: \
 \
Financial/compliance workloads \
🧾 3. Metadata & Ownership Replication \
Replicates: \
Object metadata \
ACLs (optional) \
Tags \
🔐 4. Cross-Account Replication \
Replicate to a bucket in another AWS account \
Requires: \
Bucket policies \
IAM role permissions \
🧊 5. Storage Class Change \
Destination can use a different storage class: \
Standard → Glacier \
Standard → Intelligent-Tiering \
🔄 6. Delete Marker Replication \
Optionally replicate deletes \
Important for: \
Keeping buckets in sync \
📦 7. Existing Objects Replication \
 \
By default: \
 \
Only new objects are replicated \
 \
To replicate old data: \
 \
Use S3 Batch Replication \
 \
